# 4. Use Java, Spring and Gradle multi-project

Date: 2024-02-03

## Status

Accepted

## Context

프로젝트의 효율적인 구현을 위해, 적합한 플랫폼 및 프로그래밍 언어의 선택이 필요합니다.
모노리스 아키텍처를 사용하기 때문에 단일 플랫폼(또는 프로그래밍 언어)을 선택해야합니다.

## Decision

이 프로젝트는 `자바 Java` 언어와 `스프링 Spring` 프레임워크를 기반으로 구축되며, `그레이들 Gradle`을 빌드 자동화 도구로 사용하기로 결정했습니다.

자바는 강력하고 안정적인 프로그래밍 언어로, 널리 사용되고 있으며, 스프링 프레임워크와 함께 사용될 때 개발자에게 익숙하고 효율적인 개발 환경을 제공합니다. 스프링은 속도, 단순성, 생산성을 중요시하며, 자바
애플리케이션 개발에 있어 인기 있는 프레임워크 중 하나입니다. 이러한 선택은 모듈형 모노리스 구현에 있어 우수한 사례를 제시하고자 하는 프로젝트의 목표와 부합합니다.

그레이들은 JVM 생태계에서 널리 사용되는 빌드 자동화 도구로, 다양한 개발 작업을 효율적으로 지원합니다. 특히 `다중 프로젝트 multi-project` 빌드 기능을 통해 프로젝트 내 각 모듈(하위 프로젝트)을
독립적으로 관리하고, 빌드할 수 있는 유연성을 제공합니다. 이는 모듈형 모노리스 아키텍처의 구현을 용이하게 하며, 프로젝트의 관리와 확장성을 크게 향상시킵니다.

## Consequences

- 프로젝트는 자바와 스프링 프레임워크를 사용하여 개발되어, 강력하고 유연한 백엔드 시스템을 제공합니다.
- 그레이들을 도입함으로써, 개발 프로세스의 효율성이 증가하고, 다양한 빌드 및 배포 작업이 자동화됩니다.
- 그레이들 다중 프로젝트 빌드 기능을 활용하여, 프로젝트 내 모듈 간의 의존성 관리가 간소화되고, 각 모듈의 독립적인 개발 및 테스트가 용이해집니다.
